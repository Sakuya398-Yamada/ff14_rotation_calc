# Phase 6: コードレビュー（code-reviewer）

**目的**: 実装完了後、PRを作成する前にコード品質を確認する。バグや規約違反を早期に検出し、PRの品質を上げる。

## 実行手順

1. 実装が完了したら、**`code-reviewer` サブエージェントを3個並列で起動**する
   - `Agent` ツールで `subagent_type: "code-reviewer"` を指定する
   - エージェント定義は `.claude/agents/code-reviewer.md` に集約済み
2. 各エージェントには異なる観点でレビューさせる：
   - **シンプルさ・DRY・可読性**: 「コードの簡潔さ、重複排除、可読性の観点でレビューして」
   - **バグ・機能的正確性**: 「ロジックエラー、エッジケース、バグの観点でレビューして」
   - **プロジェクト規約・アーキテクチャ**: 「CLAUDE.md と .claude/rules/*.md の規約遵守、および既存アーキテクチャとの整合性をレビューして」
3. 信頼度80以上の指摘のみ集約し、重要度順にユーザーに提示する
4. **【MCP: Playwright】** フロントエンド（React）の変更を含む場合、`browser_navigate` → `browser_snapshot` でUI動作を確認する
   - 開発サーバーが起動済みの場合のみ実行する（起動していない場合はスキップ）
   - MCP未接続時はスキップし、ユーザーに手動確認を依頼する
5. **ユーザーの判断を仰ぐ**（今すぐ修正 / 後で対応 / このまま進める）
6. ユーザーの指示に従って修正を適用する

## スキップ条件

- ユーザーが明示的にスキップを指示した場合
- ドキュメントのみの変更（`docs` ラベル）
